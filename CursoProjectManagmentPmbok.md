## Clase 1: Introducción gestión de proyectos

**¿En que consiste la gestión de proyectos?**

Es la aplicación de las herramientas, conocimientos para cumplir con los requerimientos de un proyecto.

La guia PMBOOK tiene 5 fases de proyecto y 10 areas de conocimientos.

**¿Qué es un proyecto?**

Un proyecto es un esfuerzo temporal y único que tiene un inicio y un fin para lograr un producto, servicio o resultado unico. (PMBOK,Guide, PMI)

**¿Cual es la triple restricción de un proyecto?**

![image](https://user-images.githubusercontent.com/105448434/236076538-235bdb90-2f94-4c7d-84ac-415956119e2c.png)

Alcance, Tiempo y Costo que nos determina la calidad.

**Atributos de un proyecto**

1. Tiene un proposito unico
2. Es temporal
3. Requiere recursos
4. Conlleva incertidumbre

![image](https://user-images.githubusercontent.com/105448434/236076796-8f38a45b-394a-4f9f-8be6-8bf5e8d267ed.png)

**Etapas de un proyecto**

En la fase de un proyecto siempre vamos a revisar, entradas, procesos y herramientas y salidas.

**Inicio**: Comienza el esfuerzo para generar algo. Se confirma caso de negocio, se identifican supuestos y riesgos. Objetivos medibles y se desarrolla **acta de constitución**, determinación de interesados.

**Planificación:** Se planifica el proyecto que actividades se deben ejecutar para el cumplimiento de los objetivos.
  1. Definición de alcance
  2. Identificar actividades y henerar el cronograma (linea base    tiempo, costos y alcance)
  3. Metricas de calidad
  4. Definición de roles
  5. Planificar comunicaciones
  6. Reunión de inicio del proyecto.
 
**Ejecución:** Se ejecuta el plan para general los incrementos del proyectos.

  1. Ejecutar planes y producir entregables
  2. Gestionar cambios
  3. Evaluar el desempeño 
  4. Gestionar expectativas de los interesados
  5. Resolución de conflictos.
 
Seguimiento y control.

  1. Medir desempeño segun linea base 
  2. Control integraso de cambios (Aprobar o denegar)
  
Cierro.

  1. Finalizas las actividades del proyecto
  2. Cierre financiero
  3. Aceptación de entregables
  4. Lecciones aprendidas

## Gestión de integración de un proyecto

Primero se desarrolla un:
**Caso de negocio** Aquí se selecciona el proyecto priorizando según un análisis costo-beneficio

**Acta de constitución del proyecto**

Reconoce formalmente la existencia del proyecto
1. Identifican los interesados
2. Confirman los requisitos de alto nivel
3. Objetivos y criterios de Exito
4. Riesgos

**Gestión de cambios**

En la gestión de integración se gestionan las solicitudes de cambio: 

Se deben tener contemplado: Acciones correctivas, preventivas, control de cambios y comité de control de cambios (CCB). Evaluando el impacto que tiene en la LINEA BASE.

**Proceso para realizar cambios**

1. Identificar el cambio
2. Evaluar el impacto de los cambios en nuestra linea base
3. Identificar las opciones
4. Aprobar o rechazar los cambios

## Gestión de alcance  

Es el proceso de definir que trabajo se requiere, acotar.

![image](https://user-images.githubusercontent.com/105448434/236717381-4ff41b03-4259-4355-8bc1-474f971e0192.png)

**Pasos para gestionar el alcance**

1. Recopilar requisitos
2. Definir el alcance
3. Crear la estructura de desglose de trabajo EDT
4. Validar el alcance
5. Controlar el alcance.

**Gestión de tiempo**

Establecer la duración de las actividades,como se va a gestionar y medidas correctivas y preventivas si existe desviación.

1. Definir las actividades
2. Secuenciar las actividades  (Predecesoras, dependencias, etc)
3. Estimar los recursos de las actividades
4. Estimar la duración de las actividades
5. Desarrollar el cronograma

**Métodos de estimación**

1. Utilizar juicio de expertos para estimar lo más cercano a la realidad.
2. Se puede utilizar una tecnica llamada distribucion triangular.
  
  - Se calcula un tiempo optimista, pesimista y medio. Se suman y se dividen entre 3 y se utilizan para nuestros estimados.

**Conceptos importantes**

1. Ruta critica
2. Holgura
3. Comprensión del cronograma.

## Gestión de los recursos humanos 

Como vas a involucrar personas y a gestionarlas.

**El coordinador del proyecto:**

• Determina qué recursos necesitas.
• Confirmas su disponibilidad.
• Roles y responsabilidades claramente asignados.
• Generar informes de desempeño y opciones de recompensas.

**Tipos de equipos**

• Dedicado: FULL TIME
• Medio tiempo: PART TIME o participe de MULTIPLES proyectos a la vez.
• Sociedad: Cuando multiples organizaciones trabajan en un proyecto.
• Virtual: Trabajo con personas de manera remota.

**Etapas de formación y desarrollo del equipo**
    • Formación
    • Turbulencia
    • Normalización
    • Desempeño
    • Disolución

**Gestión de conflictos**

    • Colaborar
    • Consentir
    • Eludir
    • Suavizar
    • Forzar

## Gestión de riesgos.

Se identifican y gestionan desde el inicio y se monitorean constantemente.

Riesgo: suceso probable que ocurran que afecte el proyecto. Puede existir incertidumbre de acuerdo al conocimiento que se tenga.

**Pasos para Gestionar riesgos:**

  Planificar la Gestion de los Riesgos
  
  Identificar los riesgos
  
  Realizar el análisis cualitativo
  
  Realizar el análisis cuantitativo
  
  Planificar la respuesta a los riesgos
  
  Controlar los riesgos.

**Categorias de riesgos:**

  Externos: ambientales, de gobierno, mercado, etc.
  
  Internos: cambios en tiempo, alcance, experiencia.
  
  Técnicos: cambios tecnologicos
  
  Imprevisibles: cosas que no se lograron identificar

**Estrategias de respuesta a riesgos:**

  Evitar
  
  Mitigar
  
  Transferir

# GESTIÓN DE LOS INTERESADOS.

Proceso de gestión de interesados

Identificarlos a todos (Personas que se ven impactados de una u otra forma)

Determinar sus requerimientos, expectativas e intereses. (Pueden tener diferentes niveles de requerimientos, determinar lo mínimo que deben comunicarles y que es lo mínimo esperando en ellos)

Determinar su nivel de influencia.

Planificar comunicaciones. ( Que información es relevante)

Gestionar sus expectativas, influencia y compromiso. (Los objetivos o entregables deben ser comprendidos por los interesados)

Comunicarte con ellos y controlar las comunicaciones.
